# Manual Testing Design

## 1. Test Case Structure
- **ID/Title**: Clear and unique.
- **Preconditions**: State of the system before starting.
- **Steps**: Clear actions.
- **Expected Result**: What should happen.

## 2. Exploratory Testing
- Complement scripted cases with exploratory sessions to find edge cases.

## 3. Bug Reporting
- Steps to reproduce.
- Actual vs Expected.
- Evidence (screenshots/logs).
- Severity and Priority.
